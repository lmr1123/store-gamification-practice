# Program 项目进度总结

更新时间：2026-04-28

## 1. 当前目标
- 项目定位：药店门店员工会员办卡对练系统（顾客Agent + 数字人）。
- 当前阶段目标：先确保“手机端网页可用”，尤其语音输入可稳定使用；企业微信 JS-SDK 主链路待正式上线前改造。

## 2. 已完成进度

### 2.1 后端能力
- 已新增 `POST /api/scenario-init`（场景初始化）。
- 已新增 `POST /api/customer-turn`（顾客回合，两步式：规则决策 + LLM表达）。
- 已升级 `POST /api/score` 为结构化行为评估。
- 已新增 `POST /api/transcribe`（语音转写），并支持 ASR Provider 切换：
  - `xfyun`（讯飞，默认）
  - `glm`（兼容回退）
  - 当讯飞凭据未配置时，自动回退 `glm-fallback`，保障手机端可用

### 2.2 前端能力
- AI 对练流程已接新接口（scenario-init / customer-turn / score）。
- 数字顾客头像支持情绪切换。
- 顾客提示从“数值条”改为“大白话提醒”：
  - 顾客此刻在想什么
  - 当前疑虑
  - 店员下一句建议
- 语音输入已改为“双通道”：
  - 优先：浏览器内置 SpeechRecognition（可用则用）
  - 兜底：`getUserMedia + AudioContext` 录音，上传后端转写（适配内嵌 WebView）

### 2.3 对话体验优化
- 顾客回复已做去重和多样化处理，避免连续重复“还有别的优惠吗？”
- 已补充更多顾客意图分流（规则追问、门槛确认、分享需求、离场意图等）。

### 2.4 本次新增（顾客Agent可配置化）
- 新增 `customer-agent-engine.js`：将顾客Agent能力从“写死代码”升级为“可配置运营表”。
- 已内置并可通过接口下发：
  - 会员权益事实清单（15元套券、满9.9减5、次日生效、1个月有效、全场通用）
  - 收银办卡场景槽位表（会员状态、权益说明、规则说明、流程说明、反感触发、重复风险等）
  - 决策规则表（R01-R10）
  - 反感触发词表
  - 5类顾客画像（含关注优先级与典型异议）
- `POST /api/scenario-init` 已返回 `playbook`（运营配置摘要）和完整 `persona` 元信息。
- 新增 `GET /api/customer-agent-playbook`，便于产品/运营独立查看和校对规则配置。
- `POST /api/customer-turn` 已切换到新引擎执行：
  - 先做槽位识别
  - 再做规则决策
  - 最后做自然语言表达（含一致性校验与反重复保护）
- 前端已修复画像漂移问题：`personaId` 在初始化后会持续透传到每轮对话，避免顾客角色每轮随机变化。

### 2.5 本次新增（GitHub Pages 体验修复）
- 已定位线上旧逻辑根因：`screen-ai-practice.jsx` 固定请求 `http://localhost:8899`，在 GitHub Pages 环境不可达，导致自动退回旧版 demo 回答逻辑。
- 已改为“环境自适应 API 地址”：
  - 本地 `localhost/127.0.0.1` 默认走 `http://localhost:8899`
  - GitHub Pages 默认不再强绑 localhost（避免假连接）
  - 支持通过 URL 参数 `?api=你的接口地址` 显式指定线上 API
- 已升级前端 demo 回合规则，保证即使在纯静态页（无后端）也遵循“先对齐店员话术再追问”的新逻辑，减少重复问“还有优惠吗”。

### 2.6 本次新增（对话死循环修复）
- 已定位问题：顾客会在短轮次内重复同一类问题（如“今天办卡今天能减吗”），导致体验像“死循环”。
- 已在后端顾客引擎新增“语义去重”机制：
  - 不只避免同一句话重复，也避免同一意图在近4轮内重复触发
  - 若命中重复，自动切换到下一个合理意图（规则核对 -> 流程确认 -> 异议/收口）
- 已在前端 demo 同步同样机制，保证 GitHub Pages（无后端）场景也不会重复追问同类问题。
- 修复后表现：即使店员回答不完整，顾客也会转为“换角度追问/提出异议/收口”，而不是原地重复问同一句。

### 2.7 本次新增（离场冷却规则）
- 已补“离场冷却”逻辑：顾客说出“先结账/先不办”后，若店员下一句没有新增权益或流程信息，顾客将继续收口，不会重开优惠追问。
- 本次主要修复案例：`我再想想，先把账结了。 -> 好的 -> 今天办卡今天能减吗？`，现已改为继续“先结账”类回复。
- 后端规则引擎与前端 demo 均已同步该逻辑，确保线上静态页与本地服务表现一致。

### 2.8 本次新增（意图引擎重构）
- 顾客Agent从“字段/槽位优先”升级为“主意图驱动 + 槽位约束”双轨决策：
  - 意图固定为 `value / time / risk`
  - 每轮先计算主意图，再决定顾客发言
  - 禁止仅按槽位顺序机械追问
- 新增“意图分数与阈值”机制：
  - `value/time/risk` 三类都降到阈值以下，才进入 `ready_join` 接受状态
  - 未全部解决时只会继续追问、异议或延后决定，不会提前成交
- 新增“真实业务语料自动关键词映射”：
  - 基于店长调研语料自动生成 `value/time/risk` 关键词池
  - 用于每轮意图分数动态修正
- 新增“人味触发机制”：
  - 支持 `interrupt_time`（打断要重点）与 `interrupt_risk`（打断防御）
  - 符合真实顾客非线性沟通
- 新增“文本级去重 + 意图级去重”双重防循环：
  - 避免近4轮重复同一句顾客话术
  - 避免近4轮重复同一追问意图

### 2.9 本次新增（评分与前端展示）
- `POST /api/score` 已输出并保底回填 `intent_resolution`：
  - 三类意图起止分数
  - 是否解决
  - 主意图流转轨迹
- 结果页新增“顾客意图解决情况”卡片（业务可读）：
  - 展示 3 类意图是否已解决
  - 展示疑虑变化（起始 -> 结束）
- 对练页“顾客意图提醒”升级为大白话：
  - 显示当前主意图（价值/时间/风险）
  - 显示已解决 3 类中的进展，不展示难懂技术字段

### 2.10 本次新增（流程阶段防回跳）
- 修复“已进入办理流程后又回跳到旧价值异议”的对话逻辑问题。
- 新增“流程阶段锁”：
  - 当顾客已问办理流程，且门槛/生效等核心问题已说明时，
  - 在未触发新负面信号（强推/不清晰/反感）情况下，
  - 禁止回跳到“办了会不会不划算”类旧问题。
- 后端引擎和前端 demo 均已同步该规则，防止线上/离线表现不一致。
- 同步调整异议文案：`raise_objection` 优先使用“隐私/骚扰”类风险表达，避免语义错位为“划不划算”。

### 2.11 本次新增（结账收口稳定性）
- 修复“顾客已说先不办后，又再次升级为赶时间拒绝”的不自然收口问题。
- 新增“结账信号识别”（如：一共/合计/微信/支付宝/支付）：
  - 当店员进入收银结算话术时，顾客进入“结账确认/支付”语义；
  - 不再重复输出“我赶时间，先不用了”。
- 修复优惠识别误判：结账金额（如“100元”）不再被当成“会员优惠量化信息”。
- 优化重复改写策略：仅对高重复风险意图做改写，`delay_decision` 不再被错误替换成其它意图。

### 2.12 本次新增（外部顾客Agent配置导入层）
- 已支持从外部 JSON 自动加载顾客画像与意图引擎配置，默认路径：
  - `data/customer_agents.json`
  - 也支持环境变量覆盖：`CUSTOMER_AGENTS_FILE=/abs/path/to/file.json`
- 已将你提供的数据文件纳入项目：
  - `data/customer_agents.json`
  - `data/customer_agents.README.md`
- 导入能力包括：
  - 6类顾客画像（`pos_member_*`）自动映射为系统可运行 persona
  - `intent_engine.keyword_map` 自动接管意图关键词
  - `intent_engine.thresholds.accept_all_below` 接管三意图接受阈值
  - `slot_constraints` 与 `scoring_rubric` 在 `scenario-init.playbook` 中透出给前端/运营查看
- 对话回复已支持外部 `next_reply_policy` 映射（例如 value/time/risk 高位时的典型回复）。
- 已本地验证导入生效：
  - `scenario-init` 可按 `personaId=pos_member_risk_privacy_001` 正常返回
  - `playbook.externalConfig` 可见外部文件来源与版本信息

### 2.13 本次新增（收银主线锁定与重复问题抑制增强）
- 修复“顾客已进入微信支付后，店员回复‘好的’又回到优惠追问”的问题。
- 新增 `topic_switch` 连续回合锁定：
  - 当顾客已切换到“先结账/微信支付”主线，且店员未提供新业务信息时，
  - 顾客保持支付确认语义，不再回跳“今天办卡今天能减吗？”。
- 新增支付语义识别到顾客意图分类：
  - 顾客出现“付款/支付/微信/支付宝/先结这单”等，直接归类 `topic_switch`。
- 前端每轮传参历史窗口从最近8条提升到最近20条，减少“短记忆导致旧问题回流”。

### 2.14 本次新增（入门版 LLM 顾客对练模块）
- 新增独立“入门版”训练链路（与高阶 Agent 链路并行，不互相覆盖）：
  - `POST /api/beginner-init`
  - `POST /api/beginner-turn`
  - `POST /api/beginner-feedback`
- 新增 5 类入门顾客画像（隐私顾虑/价值不认同/时间紧张/负面经验/推销反感），支持前端切换。
- 新增前端页面：`screen-ai-lite-practice.jsx`
  - 支持角色选择、最多 6 轮对练、对练结束后一键生成简版反馈。
  - 反馈输出结构为：做得好的地方 / 需要改进的地方 / 下次建议话术 / 等级。
- 首页“对练”Tab 已新增入口卡片：`AI 入门版顾客对练`，路由 `ai-lite`。
- 开发导航（DevNav）已加入 `ai-lite` 快捷入口，便于验收测试。
- 已本地接口联调通过：
  - `beginner-init` 返回画像列表与开场白
  - `beginner-turn` 返回顾客回复与对练状态
  - `beginner-feedback` 返回结构化训练反馈

- 顾客提示表达不业务化（数值难懂）已替换为业务话术提示。

- 顾客回复“与店员话术不对齐”问题已显著改善（先回答会员状态，再进入权益/规则追问）。

- 顾客重复追问同一问题导致“对话死循环”问题已修复。

- “意图未全部解决却提前 ready_join”问题已修复，改为严格阈值收敛后才接受。

## 4. 当前风险与局限
- 企业微信工作台正式环境尚未接入 JS-SDK 语音主链路（上线前必须做）。
- 讯飞 ASR 依赖凭据配置（`XFYUN_APPID/API_KEY/API_SECRET`），未配置时会失败。
- 药店专有词识别准确率仍需用真实门店语料做评测和热词优化。

## 6. 运行与配置说明（当前）
- 启动命令示例：

```bash
GLM_API_KEY=xxx \
XFYUN_APPID=xxx \
XFYUN_API_KEY=xxx \
XFYUN_API_SECRET=xxx \
ASR_PROVIDER=xfyun \
node server.js
```

- 关键环境变量：
  - `GLM_API_KEY`：顾客回复/评分能力
  - `XFYUN_APPID`、`XFYUN_API_KEY`、`XFYUN_API_SECRET`：讯飞语音转写
  - `ASR_PROVIDER`：`xfyun` 或 `glm`
