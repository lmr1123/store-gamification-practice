# Program 项目进度总结

更新时间：2026-04-27

## 1. 当前目标
- 项目定位：药店门店员工会员办卡对练系统（顾客Agent + 数字人）。
- 当前阶段目标：先确保“手机端网页可用”，尤其语音输入可稳定使用；企业微信 JS-SDK 主链路待正式上线前改造。

## 2. 已完成进度

### 2.1 后端能力
- 已新增 `POST /api/scenario-init`（场景初始化）。
- 已新增 `POST /api/customer-turn`（顾客回合，两步式：规则决策 + LLM表达）。
- 已升级 `POST /api/score` 为结构化行为评估。
- 已新增 `POST /api/transcribe`（语音转写），并支持 ASR Provider 切换：
  - `xfyun`（讯飞，默认）
  - `glm`（兼容回退）
  - 当讯飞凭据未配置时，自动回退 `glm-fallback`，保障手机端可用

### 2.2 前端能力
- AI 对练流程已接新接口（scenario-init / customer-turn / score）。
- 数字顾客头像支持情绪切换。
- 顾客提示从“数值条”改为“大白话提醒”：
  - 顾客此刻在想什么
  - 当前疑虑
  - 店员下一句建议
- 语音输入已改为“双通道”：
  - 优先：浏览器内置 SpeechRecognition（可用则用）
  - 兜底：`getUserMedia + AudioContext` 录音，上传后端转写（适配内嵌 WebView）

### 2.3 对话体验优化
- 顾客回复已做去重和多样化处理，避免连续重复“还有别的优惠吗？”
- 已补充更多顾客意图分流（规则追问、门槛确认、分享需求、离场意图等）。

### 2.4 本次新增（顾客Agent可配置化）
- 新增 `customer-agent-engine.js`：将顾客Agent能力从“写死代码”升级为“可配置运营表”。
- 已内置并可通过接口下发：
  - 会员权益事实清单（15元套券、满9.9减5、次日生效、1个月有效、全场通用）
  - 收银办卡场景槽位表（会员状态、权益说明、规则说明、流程说明、反感触发、重复风险等）
  - 决策规则表（R01-R10）
  - 反感触发词表
  - 5类顾客画像（含关注优先级与典型异议）
- `POST /api/scenario-init` 已返回 `playbook`（运营配置摘要）和完整 `persona` 元信息。
- 新增 `GET /api/customer-agent-playbook`，便于产品/运营独立查看和校对规则配置。
- `POST /api/customer-turn` 已切换到新引擎执行：
  - 先做槽位识别
  - 再做规则决策
  - 最后做自然语言表达（含一致性校验与反重复保护）
- 前端已修复画像漂移问题：`personaId` 在初始化后会持续透传到每轮对话，避免顾客角色每轮随机变化。

### 2.5 本次新增（GitHub Pages 体验修复）
- 已定位线上旧逻辑根因：`screen-ai-practice.jsx` 固定请求 `http://localhost:8899`，在 GitHub Pages 环境不可达，导致自动退回旧版 demo 回答逻辑。
- 已改为“环境自适应 API 地址”：
  - 本地 `localhost/127.0.0.1` 默认走 `http://localhost:8899`
  - GitHub Pages 默认不再强绑 localhost（避免假连接）
  - 支持通过 URL 参数 `?api=你的接口地址` 显式指定线上 API
- 已升级前端 demo 回合规则，保证即使在纯静态页（无后端）也遵循“先对齐店员话术再追问”的新逻辑，减少重复问“还有优惠吗”。

### 2.6 本次新增（对话死循环修复）
- 已定位问题：顾客会在短轮次内重复同一类问题（如“今天办卡今天能减吗”），导致体验像“死循环”。
- 已在后端顾客引擎新增“语义去重”机制：
  - 不只避免同一句话重复，也避免同一意图在近4轮内重复触发
  - 若命中重复，自动切换到下一个合理意图（规则核对 -> 流程确认 -> 异议/收口）
- 已在前端 demo 同步同样机制，保证 GitHub Pages（无后端）场景也不会重复追问同类问题。
- 修复后表现：即使店员回答不完整，顾客也会转为“换角度追问/提出异议/收口”，而不是原地重复问同一句。

## 3. 已解决问题
- 端口占用导致服务起不来（EADDRINUSE）已定位处理。
- IAB/内嵌环境访问本地回环地址不稳定，已确认局域网地址可访问。
- 顾客提示表达不业务化（数值难懂）已替换为业务话术提示。
- 顾客回复重复问题已修复。
- 顾客回复“与店员话术不对齐”问题已显著改善（先回答会员状态，再进入权益/规则追问）。
- 顾客画像每轮随机漂移（导致逻辑错乱）已修复。
- GitHub Pages 误连本地服务导致体验旧逻辑的问题已修复。
- 顾客重复追问同一问题导致“对话死循环”问题已修复。

## 4. 当前风险与局限
- 企业微信工作台正式环境尚未接入 JS-SDK 语音主链路（上线前必须做）。
- 讯飞 ASR 依赖凭据配置（`XFYUN_APPID/API_KEY/API_SECRET`），未配置时会失败。
- 药店专有词识别准确率仍需用真实门店语料做评测和热词优化。

## 6. 运行与配置说明（当前）
- 启动命令示例：

```bash
GLM_API_KEY=xxx \
XFYUN_APPID=xxx \
XFYUN_API_KEY=xxx \
XFYUN_API_SECRET=xxx \
ASR_PROVIDER=xfyun \
node server.js
```

- 关键环境变量：
  - `GLM_API_KEY`：顾客回复/评分能力
  - `XFYUN_APPID`、`XFYUN_API_KEY`、`XFYUN_API_SECRET`：讯飞语音转写
  - `ASR_PROVIDER`：`xfyun` 或 `glm`
